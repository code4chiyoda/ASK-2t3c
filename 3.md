# 風ぐるま乗り換え案内 WEB アプリケーションの有効性と課題

## 概要

風ぐるま乗り換え案内 WEB アプリケーションは、公開されている複数の情報源で示されている課題やニーズに対応する可能性があり、千代田区の地域福祉交通サービスの向上に貢献する可能性を秘めていると考えられます。以下に、有効性と課題について検討します。

## アプリケーションの有効性（テーゼ）

### 1. 利用者の利便性向上

#### 1.1 事前計画の重要性への対応

風ぐるまの本数が少ないという現状は、「令和６年第１回千代田区地域公共交通協議会 会議録」や「東京都千代田区における地域福祉交通「風ぐるま」の利用実態と今後の活用の可能性」からも明らかです。日付時刻と出発地・目的地を指定することで最適経路を表示する機能は、**利用者にとって不可欠な事前計画を支援し、限られた運行本数の中でも効率的な移動を可能にする**点で非常に有効です。

#### 1.2 わかりやすさへの貢献

「千代田区地域福祉タクシー 「風ぐるま」の見直しに関する提言書」では、見直し方針の一つとして「わかりやすく利用がしやすい『風ぐるま』の運行」が挙げられており、「路線の短縮化および分かりやすい路線」や「スムーズな乗り継ぎの実現」が提案されています。これは、既存の路線に分かりにくい点や乗り継ぎの課題があるという認識に基づいていると考えられます。また、「風ぐるまの課題解決に向けた調査検討結果 概要版 - 千代田区」では、複数のルート案が検討される中で、「ルートが複雑化する」という懸念が示されているものもあります。さらに、見直し方針の一つとして「わかりやすい運行（路線の短縮化、効率化、シンプル化）」が挙げられています。これらから、**現状、運行ルートが決してわかりやすいとは言えない事情がわかります。このアプリケーションは、まさにこの課題を直接的に解決します。**

#### 1.3 GoogleMap との差別化

「令和６年第１回千代田区地域公共交通協議会 会議録」において、事前に提出された樋口委員の意見として、「風ぐるまの改善要望についてということで、グーグルマップで経路を検索できるようにしてほしい」という具体的な要望が示されています。しかし、GoogleMap は多様な交通機関から健常な青年にとって最適と思われる経路を探索する設定が標準であり、**風ぐるまをあえて推薦する理由がありません。** 独自の乗り換え案内アプリケーションが必要です。

### 2. 情報格差の是正

#### 2.1 IT リテラシーへの配慮

IT リテラシーが不足している人向けに、経路案内を印刷して手渡せる機能は、「千代田区地域福祉計画」で課題として挙げられている**情報弱者（高齢者など）への情報提供**の強化に直接的に貢献します。デジタルツールに不慣れな区民も、IT リテラシーのある人のサポートを通じて、風ぐるまの情報を、複雑な時刻表の中から選択的に、容易に得られるようになることは、**情報格差の是正**という観点からも重要です。

### 3. 地域活性化への貢献

#### 3.1 地域情報の提供

千代田区内のガイド機能は、「千代田区地域福祉計画」が目指す**地域活動への参加促進**や、**地域とのつながりづくり**に間接的に寄与する可能性があります。区内の情報を知るきっかけを提供することで、風ぐるまの利用目的を広げ、**地域住民の生活の質向上**に貢献することが期待できます。

#### 3.2 利用者の視点に立った改善

これらの機能は、「令和６年第１回千代田区地域公共交通協議会 会議録」で示された**利用者の利便性向上への意識**や、「千代田区地域福祉タクシー 「風ぐるま」の見直しに関する提言書」で提案されている**安心して利用できる情報提供**といった方向性と合致しています。利用者の具体的なニーズに応えるアプリケーションであると言えます。

## 潜在的な課題（アンチテーゼ）

### 1. データの信頼性に関する課題

#### 1.1 情報の正確性と最新性

風ぐるまの運行ルートや時刻表は変更される可能性があるため、アプリケーションの情報が常に**最新かつ正確**である必要があります。特に、2025 年 4 月現在オープンデータとして公開されている GTFS は 2025 年 3 月に有効期限切れとなった古い情報であり、独自に編集を加えたデータを使用して対応している現状があります。これは、データの正確性に対する強い懸念となりえます。

#### 1.2 地域情報の正確性と最新性

現在、地域情報は私が手入力した情報と OpenStreetMap（ボランティアにより作成された地図）から抽出したものを提供しています。ボランティアごとに知識の偏りがあること、必ずしも正確性と最新性が保証されないことが懸念されます。**正確かつ最新のデータソースの入手が課題です。** また、ガイドのために準備している写真についても、私が個人的に交渉・撮影している現状があります。**これらのデータはオープンなものがなく、このプロジェクトは区内のオープンデータを整備する側面も持ち始めており、私個人への負担が大きくなっています。**

### 2. 既存サービスとの関係

#### 2.1 GoogleMap での検索

試してみたところ、「交通手段がバスのみかつ車いす対応」の設定をすれば、風ぐるまが表示される可能性があることがわかりました。この設定を認知しており、アクセス可能な人であれば、このアプリケーションは不要です。**この設定方法を周知させるアプローチもいいのではないでしょうか。**

#### 2.2 区の公式情報との連携

現在、一切の連携はなく運営は完全にボランティアです。アプリケーションが**区の公式情報とどのように連携または補完できるか**を計画する必要があります。重複や矛盾を避けるための調整が不可欠です。

### 3. アクセシビリティと持続可能性

#### 3.1 アクセシビリティの課題

IT リテラシーに配慮した印刷機能は重要ですが、アプリケーション自体が**すべての人が利用しやすい設計**になっているかも検討する必要があります。「千代田区地域福祉計画」でも、情報のアクセシビリティ向上が重視されています。特に、プリント・ディスアビリティを含む視覚障害者や非日本語母語話者に向けた支援機能が不足しています。WCAG（多様な特性に対応するための国際規格）や i18n（国際化: 多言語対応など）といった各種支援の充実を図っていく必要があります。

#### 3.2 維持・管理体制の課題

アプリケーションの**開発費用だけでなく、長期的な維持・管理費用**についても考慮する必要があります。誰がどのように保守・運用していくのか、持続可能な体制を示すことが求められます。完全にボランティアで運営している現状があります。**運用費を私の生活費から捻出しているということです。私は障害特性により働くことができません。生活費は貯金の切り崩しによって確保しており、収支はつねにマイナスです。現状では持続しません。** 現在は 1 時間に 10 リクエストのレート制限を設け、利用者のかたに支援金の支払いをお願いしています。

#### 3.3 利用促進策との連携

アプリケーションの存在を区民に広く知ってもらうための**周知活動**が不足しています。区の広報チャネルとの連携や、地域イベントでの紹介など、利用促進策を協議する必要があります。

### 4. アプリケーションの有効性に対する懐疑

#### 4.1 IT リテラシーが無い人にとって本当にアクセシブルか

これは Web アプリケーションで、スマートフォンや PC を持っている人にしか有効性がありません。印刷機能を通じて IT リテラシーのある他者からサポートを受けることを想定していますが、これは有効性が限定的であることを示します。時刻表が暗黙的にでもこのアプリケーションに一元化されることはあってはならず、依然として紙の時刻表は必要です。

## 結論（ジンテーゼ）

1. 風ぐるま乗り換え案内 WEB アプリケーションは、**利用者の事前計画を支援し、IT リテラシーに課題のある層への選択的な情報提供を強化し、地域情報の提供を通じて利用促進に繋がる**可能性を秘めており、千代田区の風ぐるまサービスの利便性向上に大きく貢献することが期待できます。
2. その有効性を最大限に引き出すためには、**情報の正確性・最新性の確保、区の公式情報との連携、アクセシビリティへの配慮、持続可能な維持・管理体制の構築**といった課題についても認識し、具体的な対策案を協議することが重要です。
3. このアプリケーションは、**あくまでオプションのひとつであるべきです。** 紙面からも同様の情報が得られる体制を持続、あるいは強化すべきです。
4. 限定的ではあるものの、GoogleMap で風ぐるまのルートが表示される手段がわかりました。**この設定方法を周知させること**が重要です。
