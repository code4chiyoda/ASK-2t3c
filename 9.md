# グループで仕事をするにあたって必要なこと

## 動機

[知見求む（ASK 文書）について](1.md) に記載の通り、ひとりではやっていけません。直接の当事者を招き、グループで仕事をすることになります。もし Code for Chiyoda 主導でグループを作ることが叶うのであれば、こんな風に進めたいと思っています。

## ワーキンググループ

テーマごとに同好の士が集まり、作業を行うグループをつくりましょう。作業を完了し、解散することを目的とします。

- テーマについて、関心・不安・意見がある人たち（テーマの直接の当事者が最も望ましいです）を集めます。
- 各参加者の関心・不安・意見を、互いに理解し合います。
- グループの活動の趣旨を憲章にまとめます。
- 座長（1 名か、数名の共同座長）を決めます。
- 会議で、もし決定を行わなければならない場合、大まかな合意（後述）に沿って決めます。票決はしません。

参加要件・資格審査・年会費は設けません。メールを使うのが最適です: メーリングリストを作りましょう。不足があるなら会議をしましょう。

ただし、会議をするには参加費が必要かもしれません。家賃を払うためです。千代田図書館の研修室は最適です: 図書館はあらゆる創造と文化消費のファシリテーターです（Agnoli 2016 p.70）。

## 大まかな合意

ワーキンググループは、「大まかな合意」の過程を通じて意思決定を行います。票決はしません。

Code for Chiyoda における合意では、すべての参加者が合意することは（もちろんそれが望ましいですが）必要としません。一般に、ワーキンググループでの支配的な見解が優先されます。

しかしながら、「支配的」というのは、数や粘り強さのことではありません。より一般的な意味での合意です。つまり、見解の一致です。

見解が一致するためには、なるべく全員が納得する必要があります。つまり、目の前にある手段（イチョウの木を切ろう・イチョウの木を切るな）ではなく、共通の目的（千代田区を尊厳ある街にするにはどうすればよいか？）に何度でも立ち返って、異議がなくなるまでさまざまなアイデアのテストを繰り返す必要があります。

合意は、手を挙げる・鼻歌を歌う・メールを送る・その他のワーキンググループによって（もちろん大まかな合意によって）同意された方法……などといった方法によって決定されるかもしれません。

留意: 51% の賛成は「大まかな合意」には不十分ですし、99% の賛成は大まかな合意よりも良い合意です。

大まかな合意に至ったか決定するのは、座長しだいです。

## セキュリティについての考慮事項

座長は最初に決めるべきではありません: 何よりも重要なのは互いの関心・不安・意見であって、最初の段階では、それ以外はノイズにすぎません。最初に決めると予期せぬ脆弱性が発生する可能性があります。

メールで話し合う場合、疎通の確認を取ったほうが良いかもしれません: メールはときどき届かないことがあります。

いまはその時ではありませんが、いつかスパムメールを防ぐ方法を考えたほうがいいかもしれません。
